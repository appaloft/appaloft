# Security Policy

## Supported Versions

Security fixes are applied to the latest released major version. Upgrade to the latest Appaloft
release before reporting an issue that may already be fixed.

| Version | Supported |
| --- | --- |
| 1.x latest release | Yes |
| Earlier releases | No |

## Report A Vulnerability

Do not open a public issue for a suspected vulnerability or include exploit details, secrets,
credentials, production endpoints, or customer data in public discussions.

Email [`admin@appaloft.com`](mailto:admin@appaloft.com) with:

- affected version and deployment mode;
- impact and prerequisites;
- minimal reproduction steps or proof of concept;
- whether the issue is already being exploited;
- a safe way to contact you for follow-up.

We will acknowledge the report, validate the affected boundary, coordinate a fix and disclosure
plan, and credit the reporter when requested and appropriate. Do not test against infrastructure or
accounts you do not own or have explicit permission to assess.

For the project's security architecture and credential boundaries, see
[docs/SECURITY.md](./docs/SECURITY.md).
