# Desktop Packaging Comparison

## Decision

Use `appaloft-binary-bundle` as the deployment artifact.

For a desktop wrapper, use the Tauri + sidecar binary shell in `apps/desktop`.

## Compared Options

| Option | Development | Packaging | Current measured size | Keep? |
| --- | --- | --- | --- | --- |
| Binary bundle | `bun run package:binary-bundle` then `./dist/release/appaloft-binary-bundle/run-appaloft.sh serve` | Bun `--compile` executable with embedded web assets and PGlite runtime assets | `81M` for `dist/release/appaloft-binary-bundle` on macOS arm64 | Yes, deployment default |
| Tauri + binary | `bun run package:binary-bundle` then `bun run --cwd apps/desktop dev` | Tauri app with the same `appaloft` executable prepared as a sidecar | `93M` for `apps/desktop/src-tauri/target/release/bundle/macos/Appaloft.app` on macOS arm64 | Yes, desktop shell |

## Notes

- The Tauri app is a shell only: it starts the same backend binary, opens the loopback web console,
  and exposes `window.appaloftDesktop.selectDirectory()` for the web console.
- `apps/desktop` is not wired into root `build` or `typecheck` yet because the current CI
  workflow installs Bun only. Add Rust before making Tauri part of mandatory checks.
- The Tauri sidecar input is generated by `apps/desktop/scripts/prepare-sidecar.ts`; the
  generated `src-tauri/binaries/appaloft-*` files are ignored.
- Tauri sidecar naming is target-specific. Override the detected target with
  `APPALOFT_TAURI_TARGET_TRIPLE` for cross-target preparation.

## Next Cut

1. Install Rust locally and in CI.
2. Run `bun run desktop:package`.
3. If the app launches and the native directory picker still works, wire desktop packaging into CI.
